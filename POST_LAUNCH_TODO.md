# 🚀 할러(Haler) 런칭 후 UI/UX 및 브랜딩 고도화 목록

이 문서는 할러 서비스 런칭 이후, 사용자 신뢰도를 높이고 브랜드 정체성을 완벽하게 구축하기 위해 수행해야 할 기술적 설정들을 담고 있습니다.

---

## 1. 구글 로그인 화면 'Haler' 브랜딩 완성
사용자가 구글 로그인 시 `etrz...supabase.co` 대신 `Haler`라는 이름을 가장 먼저 보게 하는 작업입니다.

### ✅ 목표
- 구글 동의 화면 중앙에 **Haler** 이름과 로고 노출
- '확인되지 않은 앱' 경고창 제거

### 🛠 수행 방법
1. **구글 클라우드 콘솔 설정**:
   - [Google Cloud Console](https://console.cloud.google.com/) 접속
   - `APIs & Services` > `OAuth consent screen` 이동
   - **[앱 게시(PUBLISH APP)]** 버튼 클릭 (테스트 상태 -> 프로덕션 전환)
2. **앱 정보 보강**:
   - 앱 로고(Haler 로고) 업로드
   - 홈페이지 주소, 개인정보처리방침, 약관 링크 입력
3. **구글 앱 인증(Verification)**:
   - 사용자 수가 늘어나면 구글에 정식 인증을 요청하여 '확인되지 않은 앱' 경고를 완전히 제거 (서치 콘솔을 통한 도메인 소유권 확인 필수)

---

## 2. 수파베이스(Supabase) 커스텀 도메인 적용 (선택 사항)
로그인 과정에서 브라우저 주소창에 나타나는 `supabase.co` 주소를 숨기고 `auth.haler.co`와 같은 자체 도메인을 사용하는 작업입니다.

### ✅ 목표
- 완벽한 화이트라벨 브랜딩 (모든 URL에서 수파베이스 이름 제거)

### 🛠 수행 방법
1. **수파베이스 플랜 업그레이드**:
   - Supabase Dashboard > Settings > Subscription에서 `Pro Plan`으로 업그레이드
2. **Custom Domain 설정**:
   - `Settings` > `General` > `Custom domains` 메뉴 활성화
   - 사용할 서브도메인(예: `auth.haler.co`) 입력 및 CNAME 레코드 설정
3. **인증 리디렉션 주소 업데이트**:
   - 새로운 커스텀 도메인에 맞춰 Google/Apple OAuth의 Redirect URI 업데이트

---

## 3. 개인정보처리방침 및 서비스 약관 페이지 구축
구글 및 애플 로그인 승인을 위해 반드시 필요한 법적 문서 페이지입니다.

### ✅ 목표
- 정식 서비스 승인을 위한 법적 요건 충족

### 🛠 수행 방법
1. **페이지 생성**: `app/privacy/page.tsx`, `app/terms/page.tsx` 생성
2. **내용 작성**: 표준 양식을 바탕으로 할러 서비스의 특성에 맞는 내용 기재
3. **푸터(Footer) 연결**: 사이트 하단에 해당 링크를 상시 노출하여 구글/애플 검수 가이드라인 준수
